# Pink Cat Audio Cutter

כלי גרפי ורוד וחמוד לחיתוך קבצי שמע עם חלון תוכנה רגיל, בלי דפדפן ובלי חלון CMD.

## מה יש עכשיו

- בחירת קובץ שמע מתוך הממשק
- מסך פעולה מעוצב לפי `menu_2.png`, עם כפתורי העלאת קובץ, חיתוך אודיו ותמלול
- נגן אודיו פנימי לתצוגה מקדימה של הקובץ שנבחר
- אזור הגדרות מסודר בטאבים: כללי, חיתוך, תמלול והיסטוריה
- בחירת תיקיית שמירה מראש
- אפשרות לתמלל קובץ מלא בלי חיתוך
- חיתוך לפי מספר מקטעים שווים
- חיתוך לפי זמנים ידניים כמו `00:00-00:30`
- ברירת מחדל לייצוא `MP3`
- בחירת פורמט פלט `OGG` / `MP3` / `WAV`
- בחירת קצב סיביות עבור `OGG` או `MP3`
- בחירת קצב דגימה
- תמלול מקומי לעברית לכל מקטע עם `faster-whisper`
- אפשרות לבחור `Azure Cloud Whisper` כמנוע תמלול חיצוני במקום המודל המקומי
- שמירת קובץ TXT לכל מקטע וקובץ תמלול מאוחד
- אפשרות לתקן תמלול בעזרת Gemini, OpenRouter או API תואם OpenAI Chat על בסיס הטקסט בלבד
- אפשרות להוסיף סיכום מקצועי בתחילת קובץ התמלול
- מצב `מקומי + תיקון API`: המודל המקומי מתמלל, ורק הטקסט נשלח ל-API לניקוי ג'יבריש, תחביר ופיסוק
- שמירת כמה מפתחות API לכל ספק בקובץ מקומי, עם ניסיון אוטומטי למפתח הבא כשיש quota/rate-limit
- תפריט `עדכון תוכנה` שמעדכן את קבצי התוכנה מ-GitHub בלי למחוק מפתחות, היסטוריה או הגדרות
- היסטוריית תמלולים בצד הממשק לפתיחת שיחות קודמות
- שמירת הגדרות אחרונות בקובץ מקומי
- חלוניות הסבר קצרות אחרי 2.5 שניות של ריחוף מעל העלאה, חיתוך ותמלול
- הדבקת מקטעי שמע ל-WhatsApp כ-MP3, בלי הודעת טקסט: הכלי מעתיק אותם דרך סייר הקבצים, בוחר `הדבק` בשורת ההודעה ומוודא שהקובץ הופיע; המשתמשת מאשרת עם Enter
- חלון סיום ממורכז ומעוצב, וחלון העברה מונפש שנשאר פתוח עד שהקבצים מזוהים בתוך WhatsApp
- בחירה בין הבוט הישראלי (`+972 55-957-1223`) לבין Transcribe Bot (`+1 413-600-0076`, שכבה חינמית מוגבלת)
- הודעות מעצימות רנדומליות בממשק, עם אפשרות כיבוי

## הפעלה

- לחצי פעמיים על `open_cat_audio_cutter.bat`

## הערות

- ברירת המחדל היא `MP3`.
- אם בוחרים `WAV`, קצב הסיביות לא בשימוש כי זה פורמט לא דחוס.
- קובצי הפלט נשמרים לפי ההגדרות שבחרת בתוך תיקיית משנה חדשה בתיקיית השמירה.
- בתמלול המקומי, בפעם הראשונה המודל עשוי לרדת מהאינטרנט. אחר כך הוא נשמר תחת `model_cache/` במחשב ורץ מקומית.
- במנוע `Azure Cloud Whisper`, הכלי מייצר קודם WAV זמני מונו 16k ואז שולח אותו ל-Azure OpenAI Whisper. זה מצב בחירה מפורשת, וברירת המחדל נשארת תמלול מקומי.
- בתפריט התמלול יש כפתור `בדוק/הורד מודל` שמכין את המודל מראש ומציג התקדמות הורדה/טעינה.
- אפשר לשמור כמה מפתחות API בקובץ `api_keys.json`, לפי ספק, או להדביק אותם ידנית בממשק.
- עבור Azure OpenAI אפשר למלא את שדות Azure בממשק אחרי בחירת `Azure Cloud Whisper`, או להשתמש בבלוק `azure_openai` מובנה עם `api_key`, `azure_endpoint`, `deployment_name`, ו-`api_version`. אפשר לשמור כמה מפתחות, והכלי ינסה את הבא אם יש quota/rate-limit או שגיאת זמינות.
- אפשר להתחיל מ-`api_keys.example.json` כתבנית, אבל לא להכניס מפתחות אמיתיים לגיט.
- `gemini_api_key.txt` עדיין נתמך למפתח Gemini יחיד לצורך תאימות לאחור.
- תיקון טקסט אחרי תמלול מקומי שולח רק את טקסט התמלול, לא את קובץ האודיו.
- סיכום מקצועי עם ספק API שולח רק את טקסט התמלול ומוסיף אותו בראש קובץ התמלול.
- ההגדרות נשמרות ב-`cat_audio_cutter_settings.json`, והיסטוריית התמלולים נשמרת ב-`cat_audio_cutter_history.json`.
- הדבקה ל-WhatsApp היא אוטומציה best-effort. הכלי מכין MP3, מציג חלון העברה מונפש, מעתיק את המקטעים דרך סייר Windows, מפעיל את חלון WhatsApp, לוחץ בשורת ההודעה ובוחר `הדבק` מתפריט הקליק הימני. חלון ההעברה נסגר רק לאחר ששינוי בחלון WhatsApp מאשר שהקבצים הופיעו. לאחר מכן המשתמשת לוחצת Enter; אם ההדבקה נכשלת הכלי מנסה גם קיצורי מקלדת ו-OGG/Opus. אם גם הניסיון הזה נכשל, הקבצים נשארים בלוח ומופיעה למשך 4 שניות הוראת הדבקה ידנית עם קישור לבוט.
- עדכון תוכנה נמצא בתפריט העליון. בהתקנה עם Git הוא מריץ עדכון Git; בהתקנה רגילה הוא מוריד ZIP מה-GitHub ומעדכן רק קבצי מקור מאושרים.
